🔹 Request

User reported that the Live Activity does not update until clicking back into the app.

🔧 Changes Made
• Updated Live Activity lifecycle handling so generic SwiftUI view disappearance no longer ends the active workout Live Activity.
• Updated the Live Activity manager to recover and update existing `ActivityKit` workout activities instead of relying only on the in-memory activity reference.
• Updated Live Activity start/end handling to clean up all active workout activities when explicitly starting a new one or ending the session.

Files modified:
• TrainState/LiveActivity/WorkoutLiveActivityManager.swift
• TrainState/Views/LiveStrengthSessionView.swift

✅ Result

Live Activity state is now kept alive across non-terminal view lifecycle changes and can be updated through existing ActivityKit activities, reducing cases where the Lock Screen or Dynamic Island state only catches up after reopening the app.
